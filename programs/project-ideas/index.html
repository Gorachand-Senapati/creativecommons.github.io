Add plain text format to CC Legal Tools

Difficulty

Medium

Size

Large (~350 hours)

Skills recommended
  • Django
  • Git
  • Python
Project Lead
  • Timid Robot

The Problem

Plain text formats for the 4.0 licenses and the CC0 public domain dedication were created using a manual process (Plaintext versions of Creative Commons licenses and CC0 - Creative Commons). Only the English versions of the legal tools were formatted as plain text. Additionally, that manual creation process was not without errors and is not sustainable.

Expected Outcome

Extend the CC Legal Tools App to generate deterministic plain text versions of the 4.0 licenses and the CC0 public domain dedication.

Contributor Tasks

The contributor will need to extend the CC Legal Tools App to generate plain text versions. The generated plain text versions must be deterministic and match the formatting of the manual versions (except where improvements are desired). Application updates must include test coverage. Stretch goals include Markdown format generation and interfacing/collaborating with SPDX.

Application Tips

  • Engage with the CC Legal Tools repositories (issues, pull requests)
  • Be helpful and welcoming (e.g. answer other contributors’ questions on Slack)
  • Applicant Guide

Resources

  • creativecommons/cc-legal-tools-app: License and public domain dedication management application for Creative Commons
  • creativecommons/cc-legal-tools-data: License and public domain dedication data for Creative Commons
  • Plaintext versions of Creative Commons 4.0 licenses - Creative Commons
  • Plaintext versions of Creative Commons licenses and CC0 - Creative Commons

Add RDF XML format to CC Legal Tools

Difficulty

High

Size

Large (~350 hours)

Skills recommended
  • Django
  • Git
  • Python
  • RDF
  • XML
Project Lead
  • Timid Robot

The Problem

The CC Legal Tools App is currently using legacy RDF files for the machine readable layer of the legal tools. Those files are out of date and contain a few errors and omissions. This technical debt also prevents future work to improve the machine readable layer of the CC Legal Tools.

Expected Outcome

Extend the CC Legal Tools App to generate deterministic RDF XML files for all of the legal tools.

Contributor Tasks

The contributor will need to extend the CC Legal Tools App to generate RDF XML files. The generated RDF XML files must be deterministic and well organized and based on current RDF specifications. Application updates must include test coverage. Stretch goals include planning any breaking changes and working on a static API.

Application Tips

  • Engage with the CC Legal Tools repositories (issues, pull requests)
  • Be helpful and welcoming (e.g. answer other contributors’ questions on Slack)
  • Applicant Guide

Resources

  • creativecommons/cc-legal-tools-app: License and public domain dedication management application for Creative Commons
  • creativecommons/cc-legal-tools-data: License and public domain dedication data for Creative Commons
  • Describing Copyright in RDF - Creative Commons Rights Expression Language

Modernize termination-of-transfer WordPress plugin

Difficulty

Medium

Size

Large (~350 hours)

Skills recommended
  • Git
  • JavaScript
  • PHP
  • WordPress
Project Lead
  • Sara

The Problem

Due to its current setup and structure it is difficult to install from the codebase directly into a WordPress installation (i.e. via Composer). Additionally, the codebase has sections that need to be checked and updated to current coding standards and best practices.

Expected Outcome

The plugin’s structure and setup follows current development standards, increasing maintainability and ease-of-use.

Contributor Tasks

The contributor will restructure the repo, adjusting any necessary code to meet current WordPress plugin development standards, and best practices for PHP and Javascript (follow appropriate ECMAScript standards), while creating a directory structure that allows the codebase to be deployed into a standard WordPress installation.

Application Tips

  • Engage with the CC Legal Tools repositories (issues, pull requests)
  • Be helpful and welcoming (e.g. answer other contributors’ questions on Slack)
  • Applicant Guide

Resources

  • creativecommons/termination-of-transfer
  • [Feature] Convert repository / codebase to a WordPress plugin that can be installed via composer #127
  • Introduction to Plugin Development
  • MDN Web Docs: ECMAScript Standards
  • PHP-FIG: PSR-1

Refactor CC meta search

Difficulty

Medium

Size

Large (~350 hours)

Skills recommended
  • CSS
  • Git
  • GitHub Pages
  • HTML
  • JavaScript
  • JSON
Project Lead
  • Sara

The Problem

The legacy CC meta search site doesn’t match the current visuals of the rest of CC web entities, and it’s built on PHP which requires a more complex hosting setup that can’t reside somewhere that supports simple static output (like GitHub pages). Additionally the code is set up in a legacy fashion, is not semantic, may contain accessibility issues, and contains broken meta searches.

Expected Outcome

Simplify and modernize the codebase and requirements of the legacy CC meta search, improve functionality, and update the visuals to match current CC standards and norms.

Contributor Tasks

Rewrite the application, maintaining its original function, as a static site on GitHub pages, using semantic, standards-compliant, and accessible HTML, CSS, and JavaScript. Update the visual aesthetic and overall appearance to better correspond with other current CC web entities.

Application Tips

  • Engage with the CC Legal Tools repositories (issues, pull requests)
  • Be helpful and welcoming (e.g. answer other contributors’ questions on Slack)
  • Applicant Guide

Resources

  • creativecommons/search: Legacy Creative Commons Search, no longer under development
  • oldsearch.creativecommons.org
  • MDN Web Docs: Semantics
  • MDN Web Docs: HTML
  • MDN Web Docs: JavaScript
  • WAVE Report for “oldsearch.creativecommons.org”
  • About GitHub Pages
